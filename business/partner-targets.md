## Partner Integration Roadmap for test-wright

The following is a list of potential SaaS/APIs to integrate with `test-wright` to enhance its functionality, expand its reach, and provide additional value to users. The integration effort, free-tier limits, and value-add for each partner are outlined below.

### 1. GitHub ([github.com](http://github.com))
- Integration Effort: Medium
- Free-Tier Limits: Unlimited public repositories, 3 private repositories for free
- Value-Add: Seamless CI/CD integration, automatic testing of pull requests, and improved developer productivity
- Rationale: GitHub is a widely-used platform for software development, making it an ideal partner for `test-wright` to reach a larger audience of developers.

### 2. Bitbucket ([bitbucket.org](http://bitbucket.org))
- Integration Effort: Medium
- Free-Tier Limits: 5 users, 5 private repositories, unlimited public repositories
- Value-Add: Similar to GitHub, Bitbucket offers CI/CD integration, automatic testing of pull requests, and improved developer productivity
- Rationale: Bitbucket is another popular platform for software development, and integrating with it will further expand the reach of `test-wright`.

### 3. Jenkins ([jenkins.io](http://jenkins.io))
- Integration Effort: High
- Free-Tier Limits: Unlimited users, unlimited builds, 1 concurrent build
- Value-Add: Jenkins is a powerful open-source automation server that can be used for building, testing, and deploying software. Integrating with Jenkins will allow `test-wright` to be used in more complex CI/CD pipelines.
- Rationale: Jenkins is widely-used in the enterprise, making it an attractive partner for `test-wright` to target.

### 4. CircleCI ([circleci.com](http://circleci.com))
- Integration Effort: Medium
- Free-Tier Limits: 100 builds per month, 1 concurrent build, 20 minutes per build
- Value-Add: CircleCI is a popular continuous integration and delivery platform that offers fast and reliable builds. Integrating with CircleCI will allow `test-wright` to be used in more efficient CI/CD pipelines.
- Rationale: CircleCI has a strong focus on speed and reliability, making it an attractive partner for `test-wright` to target.

### 5. Travis CI ([travis-ci.com](http://travis-ci.com))
- Integration Effort: Medium
- Free-Tier Limits: 100 builds per month, 1 concurrent build, 5 minutes per build
- Value-Add: Travis CI is another popular continuous integration platform that offers fast and reliable builds. Integrating with Travis CI will allow `test-wright` to be used in more efficient CI/CD pipelines.
- Rationale: Travis CI has a strong focus on simplicity and ease of use, making it an attractive partner for `test-wright` to target.

### 6. GitLab ([gitlab.com](http://gitlab.com))
- Integration Effort: Medium
- Free-Tier Limits: Unlimited users, unlimited private repositories, 2000 minutes of CI/CD per month
- Value-Add: GitLab is a complete DevOps platform that offers CI/CD, issue tracking, and project management. Integrating with GitLab will allow `test-wright` to be used in a more comprehensive development workflow.
- Rationale: GitLab is a popular alternative to GitHub and Bitbucket, making it an attractive partner for `test-wright` to target.

### 7. Semmle ([semmle.com](http://semmle.com))
- Integration Effort: High
- Free-Tier Limits: Not specified
- Value-Add: Semmle is a platform for code analysis that can help developers find and fix security vulnerabilities, code smells, and other issues. Integrating with Semmle will allow `test-wright` to provide additional value to users by offering more comprehensive code analysis.
- Rationale: Semmle has a strong focus on code quality and security, making it an attractive partner for `test-wright` to target.

### Affiliate/Revenue-Share Opportunities
- GitHub: Revenue-share available for GitHub Marketplace apps
- CircleCI: Revenue-share available for CircleCI integrations
- Travis CI: Revenue-share available for Travis CI integrations
- GitLab: Revenue-share available for GitLab integrations (via GitLab Marketplace)
